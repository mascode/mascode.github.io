How It's Made

When I decided to start a blog there was no shortage of options I could have gone with. My initial choice was going to be Wordpress but I decided to see what else I could go with. Since I've been learning programming recenly with Python, I wanted to see if there was anything out there that would let me use Python and let me create a blog at the same time. Lucky for me, there is!

After doing a bit of research I found Pelican which is a static site generator that doesn't need a database or any kind of server-side code to run. Your site is output in static HTML/CSS/JS which makes it super easy to host. Pelican supports reStructedText or Markdown markup which makes it easy to write content. There are a ton of themes available for free too! I'm currently using the Atilla theme

While Pelican is awesome, there are a few downsides to it. Since there is no database and the site is just static content this leaves out a few functions. For instance, you can't have any comments on the site without using a third party service like Disqus.

That being said I thought it would be a really good way to test out my Python skills and learn something new. I'll be going over the installation process here

Installation

I decided to go with Github Pages for my hosting. I'll be detailing that step in a bit but to summarize a little right now I had to make a new, empty repository on my github account and name it 

myusername.github.io

Which I then cloned to my desktop with

git clone https://github.com/mascode/mascode.github.io.git

This step isn't necessary if you're using a tradtional webhost, vps or developing locally.

Once the repository was cloned I navigated to the directory in a terminal to start working. Before starting off with Pelican the first thing that I did was make myself a virtual environment in Python to work in and activated it. 

python -m venv env 

./env/Scripts/activate

Once the virtual environment was activated I could now proceed with install Pelican and not have to worry about messing up my main Python installation. Installing Pelican was easy enough with

pip install "pelican[markdown]"

This installs Pelican along with Markdown support. Once pelican has been installed it has a very handy quickstart that you can use to generate the initial skeleton of your site.

Before doing that I created an empty directory called "pelican" inside my repository and navigated to it

mkdir "pelican"
cd ./pelican

Then I ran the quickstart with

pelican-quickstart

Once this command is run it will ask you a series of questions about your site and set some of the parameters for you like sitename. After the site is done generating you will see the following in your folder.

yourproject/ ├── content │ └── (pages)             # Need to make this manually ├── output                   # This is where your site is output to ├── tasks.py ├── Makefile ├── pelicanconf.py # Main settings file └── publishconf.py # Settings to use when ready to publish

Once this is done you're ready to start writing some articles and pages.

Generation

Once the virtual environment for Pelican has been setup and you've gone through the quickstart and setup your initial skeleton for the site it's time to get started with writing some articles and pages.

Quick note: Articles and Pages are handled differently in Pelican. Articles are things like blog posts and pages are content that are static in nature such as About Me or Contact pages.

Since I'm using Markdown I decided to go with a Markdown editor. For this I used Mark Text to edit and write these articles. I would have preferred using Visual Studio Code for this but I wasn't able to find a good plugin with the features that I wanted/needed. Mark Text is a wonderful piece of software for writing Markdown documents, though.

The top of your Markdown document needs to contain the metadata about your page or article. Like so

Title: My Page/Aritcle Title Date: 2020-09-09 10:20 Modified: 2020-09-10 19:30 # Optional Category: Python                 # Cannot have more than one category :( Tags: pelican, publishing Slug: my-super-post                # URL slug Authors: Alexis Metaireau, Conan Doyle             Summary: Short version for index and feeds

Here's some cotent that you wrote

Not all of these things have to be present but at the very least you're going to need Title and Author.

Once you have the metadata put in place and some content written it's time to generate your site. Remember that your markdown or restructured file has to be in the "content" direcotry.

From inside the "pelican" folder I ran the following command

pelican content pelicanconf.py

This grabs all the content that you wrote from the "content" folder and generates all the web pages into the "output" folder. If you're using a webhost the contents of this folder just need to go into your /var/www or /public_html folder on your webserver.

Hosting

Since i'm using Github Pages for hosting I had to make some slight modifications to how I generated my site. Luckily this was well [documented]((https://docs.getpelican.com/en/stable/tips.html#publishing-to-github). The way github pages works is that you put your sites content in the master branch of the repository, so what I had to do was change the output when I generated the site. 

pelican content -o .. -s peliconf.py
# -o tells the program where to output the generated files
# -s is where the pelicanconf.py is located

# The systan is as follows
# pelican /path/to/your/content/ -o /path/to output [-s path/to/your/pelicanconf.py]

By default it outputs in whatever directory you are in.

Now that I have my files being generated and output into the master branch of my directory I simply just commit and push them into the repo. Once that's done by site gets served up on mascode.github.io

With a traditional webhost, as mentioned above, the contents of the "output" folder would be put into into your /var/www or /public_html.

Some Problems

There are a few problems that I ran into

  1. Not sure if it's my theme or the way I have to generate the site but some of my links will not work properly. For example my menu doesn't show any of my pages that I made for it and I have to add them manually.

  2. Each theme works off of the same general principle but not all themes support the same features or work in the same way. This can get a bit complicated especially when it comes to customization.

  3. Each page or article can only be in one category. Pelican does not support multiple categories
